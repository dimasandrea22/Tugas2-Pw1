Tutorial HTML untuk Pemula

HTML

HTML adalah bahasa untuk membuat halaman Web dan elemen HTML sebenarnya adalah blok yang digunakan untuk membangun halaman. Pelajari dasar-dasar untuk menjadi profesional front-end.

(Artikel ini awalnya diposting di https://www.developermate.com)

HTML adalah singkatan dari HyperText Markup Language. Halaman web pertama dirilis pada tahun 1990 dan halaman tersebut hanya digunakan untuk presentasi. Hari ini web adalah bagian yang sangat penting dalam kehidupan kita sehari-hari. Anda dapat menggunakan banyak browser web yang berbeda untuk melihat halaman web, seperti: Google Chrome, Opera, Internet Explorer, dan Firefox.

Penting untuk memahami struktur halaman HTML. Tag <head> digunakan untuk judul dan meta-tag. Hanya konten di dalam Tag <body> bagian (area putih) ditampilkan oleh browser web.

Tag HTML

Tag HTML adalah nama elemen yang dikelilingi oleh kurung sudut. Tag HTML biasanya berpasangan seperti <table> dan </table>. Ingatlah untuk menggunakan aa garis miring saat Anda menulis tag akhir.

Atribut HTML

Atribut HTML digunakan untuk mengubah nilai elemen HTML. Elemen akan sering memiliki banyak atribut.

HTML <id> atribut mendefinisikan id unik untuk elemen HTML. Id harus unik dalam dokumen HTML. Anda dapat menggunakan <id> untuk memilih elemen HTML, maka Anda dapat membaca atau memanipulasi elemen tersebut dengan CSS dan JavaScript.

HTML

Elemen HTML juga bisa disebut blok bangunan. Elemen-elemen ini digunakan untuk membangun halaman HTML. Elemen-elemen tersebut terdiri dari tag awal dan tag akhir dan di antara tag tersebut kami menempatkan teks atau konten grafis.

Elemen tingkat Blok HTML

Elemen level blok selalu dimulai pada baris baru. Jenis elemen ini juga selalu menggunakan lebar penuh yang tersedia di halaman. Artinya memanjang sejauh mungkin ke kiri dan sejauh mungkin ke kanan.

Elemen Sebaris HTML

Tidak seperti elemen tingkat Blok, elemen Sebaris tidak dimulai pada baris baru. Selain itu tidak mengambil seluruh lebar, tetapi lebar sebanyak yang diperlukan.

Elemen HTML Kosong

Ada juga elemen HTML yang tidak memiliki konten dan ini disebut elemen kosong. Elemen kosong yang sering digunakan adalah elemen <br>. Ini adalah item kosong yang tidak memiliki konten.

Huruf kecil atau huruf besar?

Meskipun tidak salah dan sebenarnya sepenuhnya valid untuk menggunakan huruf besar untuk tag dan atribut saat menulis HTML, konvensinya adalah untuk selalu menggunakan huruf kecil. Ini untuk alasan historis dan kembali ke hari-hari XHTML di mana percakapan benar-benar ketat dan di mana huruf besar tidak valid.

Judul HTML

HTML <h1> - <h6> elemen mewakili enam tingkat terpisah dari judul bagian. Bagian tertinggi adalah level <h1> dan bagian terendah adalah <h6>. Anda juga bisa mengatakan itu <h1> memiliki kepentingan tertinggi dan <h6> memiliki kepentingan terendah.

Judul tidak hanya penting untuk mengubah tampilan halaman. Judul juga penting dalam hal cara kerja mesin telusur. Untuk mengindeks struktur dan konten halaman web, mesin pencari menggunakan heading.

Pengguna sering kali memeriksa halaman Anda dengan cepat hanya dengan membaca judulnya. Penting untuk menggunakan heading untuk melihat struktur dokumen.

Paragraf HTML

Elemen HTML <p> mendefinisikan sebuah paragraf. Tag paragraf menempatkan baris kosong di atas dan di bawah teks.

Tautan HTML

Tautan HTML digunakan untuk menghubungkan halaman satu sama lain. Tautan HTML juga disebut hyperlink. Sebuah situs web terdiri dari beberapa halaman dan Anda harus menautkan halaman tersebut bersama-sama. Tautan memudahkan pengguna untuk menavigasi antar halaman di situs web Anda dan bahkan ke situs web lain.

Tabel HTML

Elemen HTML <table> sedang digunakan untuk mewakili data tabel. Tabel data adalah informasi yang dapat kita sajikan dalam tabel dua dimensi. Tabel tersebut terdiri dari sejumlah baris dan sejumlah kolom sel yang berisi data.

Dalam HTML ada tiga cara berbeda untuk mendefinisikan daftar informasi. Semua daftar yang ada harus berisi setidaknya satu elemen daftar, tetapi juga dapat berisi beberapa elemen daftar.

Teks Pemformatan HTML

Untuk dapat memformat elemen teks ada baiknya untuk mengetahui opsi yang tersedia untuk mengatur format teks dalam HTML dan XHTML. Elemen pemformatan dirancang dan digunakan untuk menampilkan berbagai jenis teks.

Elemen Frase Komputer berguna untuk mendokumentasikan pengembangan kode komputer dan interaksi pengguna. Ini dilakukan dengan membedakan kode sumber <code>, variabel <var>, masukan pengguna <kbd> dan terminal atau keluaran serupa lainnya <samp>.

Karakter yang dicadangkan

Sedikit teks (string) yang dimulai dengan ampersand (&) dan diakhiri dengan titik koma (;) disebut entitas HTML. Ini sebagian besar digunakan untuk menampilkan karakter yang dipesan. Potongan teks ini akan ditafsirkan sebagai kode HTML dan karakter yang tidak terlihat. Ini juga berguna jika Anda ingin menulis karakter yang sulit diketik pada keyboard biasa.

Beberapa karakter khusus dan dicadangkan untuk digunakan dalam HTML. Itu berarti bahwa karakter ini akan diuraikan sebagai kode HTML oleh browser. Jadi ketika misalnya tanda kurang dari (<) digunakan dalam kode, browser Anda akan menafsirkan teks yang mengikutinya sebagai tag.

Komentar HTML

Komentar adalah bagian dari kode yang diabaikan oleh browser. Untuk menjelaskan dan menentukan bagian bagian dari dokumen kepada orang lain yang melihat kode, sebaiknya tambahkan komentar ke kode HTML Anda. Terutama dalam dokumen yang kompleks. Untuk meningkatkan keterbacaan kode dan membantu orang lain memahami kode, maka komentar sangat berguna.

Teks dan konten ditempatkan di antara ini <! - ... -> tag ditangani sebagai komentar dan akan diabaikan sepenuhnya oleh browser.

Tata Letak HTML

Saat Anda mengembangkan halaman web, sangat penting untuk memikirkan tata letak dan seperti apa tampilan halaman itu. Ini penting untuk seluruh pengalaman pengguna dan untuk keramahan pengguna halaman.

Saat ini, sudah umum dan sangat disarankan untuk menggunakan teknologi modern seperti CSS , JavaScript dan Angular untuk membuat halaman. Tetapi juga memungkinkan untuk membuat halaman hanya dengan HTML .

Iframe HTML

Elemen Bingkai Sebaris HTML, <iframe>, menyematkan halaman HTML lain ke halaman saat ini. Itu <iframe>dapat muncul di mana saja dalam dokumen HTML. Itu <iframe>dapat memiliki bilah gulir dan batas yang terpisah.

Gambar HTML

Di masa kecil web, hanya ada teks dan tidak ada gambar. Untungnya, tidak butuh waktu lama sebelum kami mendapat kesempatan untuk menyematkan gambar dan jenis konten menarik lainnya. Gambar tidak hanya bagus untuk dilihat tetapi juga penting digunakan untuk menjelaskan konsep kompleks dengan mudah di halaman web Anda.

Warna HTML

Warna HTML ditentukan menggunakan kata kunci nama warna yang ditentukan atau nilai RGB, HEX, HSL, RGBA, HSLA. Warna Hue adalah kata lain dari warna.

Misalnya warna hijau memiliki kode warna HEX #00FF00, yaitu juga RGB(0,255,0), yaitu '0' merah, '255' hijau, dan '0' biru. Kode warna dapat digunakan untuk mengubah warna teks dan latar belakang pada halaman web.

Nama Warna HTML

Kata kunci Nama Warna adalah pengidentifikasi peka huruf besar/kecil yang dapat digunakan sebagai pengganti RGB, HEX dll dan Nama Warna mewakili warna tertentu, seperti hijau/Hijau, biru/Biru, kuning/Kuning, atau lightseagreen/LightSeaGreen.

Class HTML

Gunakan HTML class untuk menentukan gaya yang sama untuk satu atau banyak elemen di halaman web Anda. Semua elemen HTML dengan yang sama class atribut name akan mendapatkan gaya dan format yang sama.

Sulit dan memakan waktu untuk menentukan gaya yang sama ke banyak elemen menggunakan gaya sebaris untuk setiap elemen. Kelas HTML adalah atribut yang sering digunakan saat membangun situs web yang menarik. Kelas akan membantu Anda menghabiskan lebih sedikit waktu untuk menulis dan mengkhawatirkan gaya seperti font dan warna.

Style HTML

HTML <style>-element berisi informasi tentang gaya dokumen atau bagian dari dokumen. Dalam <style>-element, Anda menemukan CSS, yang diterapkan ke konten. Tag <Style>-elemen dapat ditempatkan di dokumen <head>atau <body>tetapi biasanya disarankan agar gaya ditempatkan di <head>. Lebih baik lagi adalah menempatkan gaya di lembar gaya eksternal dan merujuknya dengan <link> elemen.

Ketika beberapa <style>- dan <link> elemen disertakan dalam dokumen Anda, mereka diterapkan ke DOM dalam urutan yang sama seperti yang disertakan dalam dokumen.

File Path HTML

File Path menentukan lokasi file. File Path seperti alamat ke file dan membantu browser web untuk menemukan file. Jalur file digunakan untuk menautkan file eksternal seperti gambar, video, lembar gaya, JavaScript , menampilkan halaman web lain, dll.
Ada dua jenis File Path: File Path Absolut dan File Path Relatif.

Elemen Formulir HTML

HTML <form> elemen digunakan untuk mengumpulkan input pengguna dan mengirimkannya kembali ke server.
Tag <form> adalah elemen tingkat blok dan mendefinisikan bagian interaktif dari halaman web. Semua kontrol di dalam <form> akan dikirim kembali ke server.

Elemen HTML mendefinisikan berbagai bagian halaman HTML. Elemen HTML didefinisikan oleh tag awal dan biasanya memiliki tag akhir. Sebagian besar elemen HTML memiliki konten di antara tag awal dan tag akhir.

<tagname>Element content...</tagname>

Jenis Input HTML

Dalam HTML, salah satu elemen yang paling kuat dan kompleks adalah <Input>elemen. Alasannya adalah banyaknya kombinasi tipe dan atribut input. Terlepas dari jenisnya, semua <input>-elements didasarkan pada antarmuka HTMLInputElement dan itu berarti mereka semua berbagi set atribut yang sama persis. Namun terlepas dari itu, tidak semua atribut akan berfungsi dengan baik untuk semua tipe input dan beberapa tipe input hanya mendukung beberapa atribut ini.

HTML <input> elemen digunakan untuk membuat kontrol yang interaktif untuk formulir berbasis web. Ini digunakan untuk menerima data dari pengguna. Ada banyak jenis data masukan dan widget kontrol untuk berbagai jenis perangkat.

Pengkodean URL HTML

URL Encodingadalah proses mengubah string menjadi format yang valid yang dapat ditransmisikan melalui Internet. URL harus menggunakan set karakter ASCII. Karakter di luar set ASCII harus dikonversi agar dapat ditransmisikan melalui Internet.

URL encodingmenggantikan karakter non-ASCII dengan "%" diikuti dengan digit heksadesimal, Anda juga dapat menggunakan istilah penyandian persen untuk proses ini.

Metode HTML untuk Rest Service

Ada banyak metode HTTP yang tersedia. Pernahkah Anda bertanya-tanya apa perbedaan antara GETdan POSTpermintaan? Kapan Anda harus menggunakan PUTdan DELETE?

Lanjutkan membaca untuk mendapatkan pemahaman dasar tentang layanan RESTful dan berbagai metode HTTP, juga disebut verba HTTP. Memahami metode HTTP penting ketika bekerja dengan API ini.

Sebagian besar layanan REST berbasis CRUD. Pola CRUD didasarkan pada protokol HTTP. Layanan CRUD mendukung metode POST, GET, PUTdan DELETE.

Metode GET HTTP

GET request adalah metode HTTP yang paling banyak digunakan. Metode GET digunakan untuk mengambil data dari server. Misalnya, Anda memiliki API dengan a /carstitik akhir. Membuat permintaan GET ke titik akhir itu akan mengembalikan daftar semua mobil yang tersedia.

Karena permintaan GET hanya meminta data dan tidak mengubah sumber daya apa pun, itu dianggap sebagai metode yang aman.

Metode POST HTTP

POST request digunakan untuk mengirim data ke server untuk membuat sumber daya. Data yang dikirim disimpan di badan permintaan permintaan HTTP.

Contoh paling sederhana adalah formulir kontak di situs web. Saat Anda mengisi input dalam formulir dan menekan Send , data tersebut dimasukkan ke badan respons permintaan dan dikirim ke server. Ini mungkin JSON, XML, atau parameter kueri (ada banyak format lain, tetapi ini adalah yang paling umum).

Metode PUT HTTP

PUT request digunakan untuk mengirim data ke server, digunakan untuk memperbarui sumber daya. Memanggil permintaan PUT yang sama beberapa kali akan selalu menghasilkan hasil yang sama. Sebaliknya, memanggil permintaan POST berulang kali akan memiliki efek samping membuat sumber daya yang sama beberapa kali.

Metode DELETE HTTP

DELETE metode akan menghapus sumber daya. Metode ini adalah salah satu yang lebih umum di RESTful API, jadi ada baiknya untuk mengetahui cara kerjanya.

Jika mobil baru dibuat dengan request POST ke /cars, dan dapat diambil dengan GET request untuk /cars/{{id}}, kemudian DELETE request untuk /cars/{{id}} akan benar-benar menghapus mobil itu.

Kode Status HTTP HTML

REST API menggunakan kode Status HTTP dalam pesan respons HTTP untuk memberi tahu klien tentang permintaan mereka yang dibuat ke server.
Digit pertama dari kode status menentukan salah satu dari lima kelas standar tanggapan.
